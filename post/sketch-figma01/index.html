一分钟了解Sketch落后于Figma的部分

2019-09-07 1 min read # UI # sketch # figma

作为sketch用户，你一定对sketch组件（symbols）一定不陌生，在一定程度上提高设计一致性和设计效率。目前来看sketch工具对图层样式，字体样式，组件管理有着比较强的逻辑。特别是在sketch更新55.2后，可以将组件内的图层样式/字体样式抽离出来，方便于定义不同的样式需求。

但由于sketch的字体样式要求对细节定义。它定义字体规则的时候会详细到，字体类型，字体大小，字体对齐方式，字间距，行高等，只能通过定义多个字体样式来进行管理。不够灵活。

庆幸的是字体样式的管理是可以通过命名来进行分组（图层样式，组件亦如此）的例如：

大标题/红色/左对齐

类似这种，我们可以根据分组进行寻找。但是效率实在太低。

figma在这点上，就做的非常的宽容。

只定义字体类型，字号即可。关于对齐方式，字体颜色可以自行调整。可以说对设计非常宽容。或许不同的设计师的工作流是不一样的。但是我觉得我在使用figma的字体样式时候，就有一种亲切感。未来sketch或许也会向这个方向尝试吧。

期待更好的设计工具。或许我就是一个工具控
